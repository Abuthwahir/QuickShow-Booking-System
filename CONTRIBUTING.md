# Contributing to QuickShow Booking System

Thank you for showing interest in contributing to the QuickShow Booking System! We welcome all contributions from bug fixes to new feature implementations.

## How to Contribute

### 1. Reporting Bugs
* Search existing issues or repository logs to verify the bug hasn't been reported.
* Open an issue with a clear description, steps to reproduce, and screenshots if applicable.

### 2. Proposing Features
* Open an issue to discuss your proposed feature before starting work.
* Explain the use case and implementation outline.

### 3. Pull Requests
1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/your-feature-name`.
3. Follow the code quality guidelines and make sure lint checks pass.
4. Commit your changes with clear commit messages.
5. Push to your branch: `git push origin feature/your-feature-name`.
6. Open a Pull Request against the `main` branch.

Please be respectful and constructive in all community interactions.
